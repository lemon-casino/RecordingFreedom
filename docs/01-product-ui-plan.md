# 01. 胶囊托盘工具窗口与产品 UI 计划

## 第一阶段范围

第一阶段先做可运行的 UI Shell，并绑定稳定的服务接口与 mock adapter。它不是临时方案：所有组件、状态名和接口都按后续真实录制接入设计。

第一阶段必须包含：

- 胶囊录制工具窗口。
- 屏幕 / 窗口 / 程序三类源选择入口。
- 系统声音、麦克风、麦克风降噪、摄像头开关和设备菜单。
- 开始录制、暂停/继续、结束录制三个主操作。
- 当前录制状态、计时器、音频电平、存储位置提示。
- 语言选择。
- 设置入口与独立设置窗口 UI。

## 托盘行为

胶囊工具窗口必须作为托盘入口的一等能力，而不是普通主窗口：

- 系统托盘图标点击：显示或隐藏胶囊录制窗口。
- 系统托盘右键菜单：显示录制窗口、隐藏录制窗口、退出 RecordingFreedom。
- 胶囊窗口关闭按钮：隐藏窗口，不直接退出应用。
- Escape：隐藏胶囊窗口。
- 失焦：隐藏胶囊窗口，让它像录制控制浮窗一样轻量收起。
- Windows：胶囊窗口不显示在任务栏。
- macOS：使用 accessory activation policy，托盘为主要入口；正式发布时再根据编辑器窗口需要恢复 Dock/菜单策略。

## 胶囊窗口信息架构

默认横向胶囊从左到右：

1. 拖拽把手。
2. 源选择胶囊段：显示当前源类型和名称。
3. 音频胶囊段：系统声音、麦克风、降噪、电平。
4. 摄像头胶囊段：摄像头开关和设备。
5. 主录制按钮：未录制为开始，录制中为结束。
6. 录制计时器：录制中显示。
7. 暂停/继续按钮：录制中显示。
8. 语言按钮。
9. 设置按钮。

紧凑竖向胶囊用于停靠屏幕左右边缘，保留相同功能顺序，但隐藏长文本，只保留 tooltip 和菜单标题。

## 源选择体验

源选择弹窗必须区分三类目标：

- `Screen`：整块显示器，显示屏幕编号、分辨率、刷新率。
- `Window`：单个窗口，显示应用名、窗口标题、缩略图。
- `Application`：按程序分组，选择程序后默认录制该程序当前主窗口；多窗口程序允许展开选择窗口。

程序识别不是单独的录制 API，而是源枚举层能力：先识别进程/应用，再映射到可录窗口或平台支持的 app capture target。

## 音频菜单

音频菜单包含：

- 系统声音开关。
- 系统声音设备选择。
- 麦克风开关。
- 麦克风设备选择。
- 麦克风降噪开关，默认开启。
- 麦克风增益滑块，默认 `1.0`，高级设置中可调整。
- 输入电平 meter，仅在非录制或录制中轻量显示。

降噪开关只影响麦克风，不处理系统声音。菜单必须显示当前算法状态：`RNNoise`、`Off`、或 `Unavailable`。

## 摄像头菜单

摄像头菜单第一阶段只做录制 sidecar 选择，不做完整画中画编辑：

- 摄像头开关。
- 摄像头设备选择。
- 分辨率预设：720p、1080p、Source。
- 帧率预设：30、60。
- 预览缩略图。
- PIP preset 选择：右下角、左下角、自由布局、关闭。

录制时摄像头写为独立 sidecar，并在 manifest 记录 `webcamStartOffsetMs` 和默认 PIP 布局。

## 视觉设计

设计关键词：现代、轻量、时尚、专业、胶囊、录制工具、创作者效率。

设计系统校准：

- 产品定位按“桌面录制工具 + 创作者效率”处理，不做营销页式 hero 和装饰卡片堆叠。
- 使用高对比、克制留白和清晰状态色表达“正在录制 / 暂停 / 可用 / 不可用”。
- 胶囊窗口保持信息密度，设置窗口承载复杂项；避免把高级录制、诊断和发布信息塞回浮窗。

视觉规则：

- 主背景：深石墨 `#101316`，浮层 `#171B20`。
- 主状态色：teal `#0D9488`，用于可用、已选择、音频正常。
- 录制色：coral/red `#F04438`，只用于录制状态和结束按钮。
- 暂停色：amber `#F59E0B`。
- 文本：主文本接近白色，次文本不低于 WCAG 3:1。
- 边框：半透明冷灰，不使用旧项目的大面积玫红。
- 透明与 blur 必须克制，不能影响可读性。
- 图标优先使用 `lucide-react`，按钮内不使用 emoji。

动效规则：

- hover/press/focus 150-220ms。
- 菜单进入使用 opacity + transform，不动画 width、height、top、left。
- 录制中只允许录制点轻微 pulse，不能有持续装饰动画。
- 必须支持 `prefers-reduced-motion: reduce`。

## 设置窗口

设置窗口使用独立 Wails window，不能塞进透明胶囊窗口。设置分类：

- General：语言、启动行为、最小化到托盘。
- Recording：默认保存位置、质量、FPS、码率、录制包保留策略。
- Source：默认源类型、鼠标光标策略。
- Audio：系统声音、麦克风、降噪、增益、采样率诊断。
- Camera：默认摄像头、分辨率、PIP 默认布局。
- Storage：显示当前真实 `data/video` 路径、应用数据根目录、缓存清理、空间提醒。
- Advanced：诊断日志、实验性 Linux 录制、FFmpeg 路径。

## UI 验收

- 胶囊窗口在 1280x720、1440x900、1920x1080、4K 下不重叠、不裁切。
- 停靠四边时菜单仍可见。
- 所有按钮有 tooltip、aria-label 和键盘 focus ring。
- 录制、暂停、结束三种状态有明确颜色和图标差异。
- 语言切换后胶囊不因文本变化撑破布局。
- 设置窗口可独立打开、关闭，并返回焦点到触发按钮。
- 设置窗口必须显示后端返回的真实 `data/video` 路径；浏览器预览可以显示 `data/video` fallback。
- 设置窗口可以修改应用数据根目录；修改后仍必须把录制包写入新根目录下的 `data/video`，不能把录制包直接散落到用户选择的裸目录。
- 设置窗口中的质量、FPS、录制光标和倒计时会保存到设置，并随开始录制请求写入 manifest。
- 音频菜单中的系统声音设备和麦克风设备会保存到设置，并随开始录制请求写入 manifest；关闭的音频流不会保留旧 device id。
- 摄像头菜单中的 PIP preset 会保存到设置，并随开始录制请求写入 manifest；摄像头关闭时录制 manifest 使用 `off`。
